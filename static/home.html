DevOpsified the Go Application

This project demonstrates the complete **DevOps** lifecycle for a Go application, from development to deployment in a **cloud-native environment**. By leveraging modern **containerization**, **CI/CD** practices, and **Kubernetes**, this project automates the build, testing, and deployment processes, ensuring a fast and reliable workflow.

Project Overview

The goal of this project was to containerize a simple **Go** application using **Docker** and set up a **Kubernetes** environment for orchestration. The application is built and deployed using **Amazon EKS (Elastic Kubernetes Service)** and automated CI/CD workflows with **GitHub Actions** and **ArgoCD** for continuous delivery.

Key Technologies Used

  • Go: Backend application written in Go.
  • Docker: **Multistage Dockerfile** for containerizing the Go application.
  • Kubernetes: For managing the deployment and scaling of the application in a cloud environment.
  • Amazon EKS: A fully managed Kubernetes service for deploying the application in the cloud.
  • GitHub Actions: Used for building and testing the application automatically on every commit.
  • ArgoCD: A declarative, GitOps continuous delivery tool to automate deployment on Kubernetes.
  • Helm: Used for managing Kubernetes packages, providing an easy way to define, install, and upgrade applications.
  • Ingress Controller: Set up for routing and load balancing of external traffic to the application.

Deployment Process

The entire deployment process is automated and consists of several stages:

  1. Build & Test: The application is built using a **Multistage Dockerfile** to ensure the smallest possible image size. Unit tests are run in the CI pipeline using **GitHub Actions**.
  2. Containerization: The application is containerized using Docker, with multi-stage builds to separate the build environment from the final runtime environment.
  3. Deployment on Kubernetes: The Kubernetes deployment, service, and ingress resources are defined using YAML files. The application is then deployed to **Amazon EKS** with Helm charts, ensuring scalability and ease of management.

  5. Load Balancing: An **Ingress Controller** is used to manage external traffic and distribute it across multiple instances of the application, providing load balancing and high availability.

Conclusion

This project showcases the power of modern DevOps practices and tools to automate and streamline the software development and deployment lifecycle. The use of Docker, Kubernetes, and cloud-native services such as Amazon EKS makes it highly scalable and efficient. Additionally, integrating CI/CD pipelines with GitHub Actions and ArgoCD allows for a seamless, automated deployment process, ensuring the application is always up-to-date with minimal manual intervention.
